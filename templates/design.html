Fase de Diseño del Proyecto

En la fase de diseño, se tradujeron los requisitos de la planificación en un diseño técnico detallado. Esto incluyó la definición de los modelos de datos (`models.py`), la estructura de las operaciones de negocio (`operations.py`) y el diseño de la interfaz de usuario.

Aspectos destacados del diseño:

  • Modelos de Datos: Se definieron las clases `Transaction`, `MarketPrice` y `Player` usando `dataclasses` para una estructura clara y eficiente.
  • Lógica de Negocio: Se centralizaron las operaciones de gestión de datos (añadir, obtener, actualizar, eliminar) en la clase `GTAOnlineOperations` para mantener el código modular y limpio.
  • Diseño de API: Se definieron los endpoints de la API RESTful utilizando FastAPI, incluyendo validación de datos con Pydantic.
  • Diseño de Interfaz de Usuario (UI): Se crearon las plantillas HTML (`templates/`) con Jinja2 para las distintas vistas (añadir/ver ítems, jugadores, transacciones, análisis), siguiendo un diseño responsivo y fácil de usar. Se utilizaron formularios para la interacción del usuario y tablas para la visualización de datos.
  • Almacenamiento de Datos: Se optó por archivos CSV y JSON para un almacenamiento simple y persistente de los datos simulados.
  • Manejo de Errores: Se implementó un manejo robusto de excepciones para proporcionar retroalimentación útil tanto al usuario como al desarrollador.

El diseño se enfocó en la modularidad, la facilidad de mantenimiento y la escalabilidad futura del sistema.
